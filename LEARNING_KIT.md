## Applied Notes

- The State Engine is a deterministic output layer, not a decision layer.
- `risk_level` is mapped directly to `state` using the locked table.
- `anomaly_flag=True` is the only override and always produces `CRITICAL`.
- `trace_id` is never generated by the State Engine.
- User-facing explanation remains owned by Mitra; State Engine does not rewrite explanation.
- The final shared batch verifies trace continuity using Nupur's signal/perception proof and State Engine live HTTP logs.
